# NumLock 설계 문서

게임 규칙은 `RULES.md` 참고. 이 문서는 아키텍처/정산/UI 등 구현 관련 결정사항.

## 아키텍처

- 훌라와 완전히 별도 앱 (로그인 시스템 등 새로 구성, 계정/DB 공유 없음)
- 4명 고정 인원
- 훌라처럼 싱글모드(AI 3명) + 멀티모드 지원
- AI는 계산 기반 휴리스틱으로 구현 예정 (LLM/API 불필요 — Qwixx는 감/대화가 아니라 계산 문제라 판단하기 쉬움)

## 재화 시스템

- 훌라 계정과 전혀 안 엮임 — NumLock 전용 포인트를 새로 만듦
- 훌라처럼 **싱글 포인트 / 멀티 포인트를 별도 풀로 관리** (설정·랭킹 화면에서 싱글/멀티를 나눠 보여주기로 하면서 자연스럽게 정해짐)
- **충전 방식 (확정, 훌라와 동일) — 앞서 "충전 없음"이라 했던 결정 번복**: 싱글 포인트는 **신규 유저 초기값 100점**, **0점일 때만 충전 가능**, **24시간 쿨다운**, **충전량 100점 고정**(훌라 싱글모드와 정확히 동일한 수치). 멀티 포인트의 초기값/충전량은 멀티모드 설계할 때(나중) 같이 정하기로 미룸

## 정산 공식

훌라와 달리 **제로섬 아님** — 진 사람이 이긴 사람한테 돈을 주는 구조가 아니라, 다들 각자 성과만큼 포인트를 얻어가는 방식(판마다 시스템에서 포인트가 새로 생겨남).

- **기본 정산**: 각자 자기 최종 점수 = 그대로 포인트 (1점 = 1포인트)
- **1등 추가 보너스**: (1등 최종점수 − 2등 최종점수)를 1등한테 추가로 지급
  - 1등-꼴등 차이가 아니라 **1등-2등 차이**로 하기로 함 — 이유: 꼴등이 벌점 몰려서 유독 낮게 나온 경우, 1등이 2등이랑 막상막하로 이겼어도 보너스가 과하게 커지는 왜곡이 생김. 1등-2등 차이가 "1등 자리를 얼마나 확실히 지켰는지"를 더 정확히 반영함
  - **공동 1등(동점)일 경우**: 보너스 없이 그냥 각자 자기 점수 그대로만 지급 (1등-2등 차이 계산 자체가 무의미해지므로)

## 로그인/계정 시스템 (확정)

훌라와 동일한 방식 — 비밀번호 없이 각자 자기 코드만 입력해서 로그인. 계정 DB는 훌라와 완전히 별개(재화 시스템 섹션 참고).

## AI 휴리스틱 (확정)

사람처럼 느껴지게 하려고 전부 **확률 기반**으로 설계(하드 threshold 지양). AI 3명한테는 성향 차이를 줘서(확률 곡선 기울기를 살짝씩 다르게) 매번 똑같이 행동하지 않게 함.

1. **숫자 체크 여부**: 지금까지 체크한 숫자와 이번 후보 숫자 사이의 "간격"이 클수록 체크 확률이 낮아지는 부드러운 확률 곡선 (예: 간격1→90%, 간격2→75%, 간격3→55%, 간격5→25%, 간격8+→5%). 하드 컷오프 없이 연속적으로.
2. **턴 내 기회 소진에 따른 보정**: 내 턴(차례인 플레이어)일 때, 공용 행동(1차 기회)에서 패스했다면, 추가 행동(2차 기회=이번 턴 마지막 기회)에서는 확률을 크게 끌어올림 — 페널티 회피 목적. "내 턴이라 다르다"가 아니라 "이번 턴에 남은 기회가 줄어들수록 절박해진다"는 개념.
3. **줄 잠그기 여부**: 조건(5칸 이상 체크) 충족 시 무조건 잠그지 않고, 이것도 확률 기반. **두 요인을 합산**:
   - 지금까지 그 줄에 체크된 칸 수 (많이 채웠을수록 잠금 확률 ↑, 예: 5칸=40%, 8칸+=85%)
   - 다른 플레이어들과 비교한 현재 순위/점수 격차 (앞서고 있으면 확률 ↑ = 빨리 끝내려 함, 뒤처지고 있으면 확률 ↓ = 판을 끌어서 역전 노림)
4. 별도의 "가끔 일부러 나쁜 선택" 로직은 불필요 — 1번의 확률 기반 판단 자체가 이미 자연스러운 불완전함을 만들어줌 (기각됨)

## 공용 행동 타이밍 (확정)

훌라의 땡큐 같은 실시간 경쟁 방식 아님 — 흰 주사위 합은 **먼저 잡는 사람이 임자인 자원이 아니라**(내가 체크한다고 남이 그 숫자를 못 쓰는 게 아님), 각자 독립적으로 "이 숫자 쓸지 말지" 결정하는 개인 선택. 그래서 실시간 클릭 경쟁 구조는 불필요. 다만 게임이 안 늘어지게 짧은 제한시간(구체 수치는 미정, 예: 10초 정도)은 둘 예정.

## 체크 실수/취소 (확정)

한 번 체크하면 되돌릴 수 없음 (훌라의 등록처럼 되돌리기 없이 확정).

## 턴 순서 (확정)

이전 판 최종 순위(1등→2등→3등→4등) 그대로 다음 판 턴 순서가 됨. 훌라처럼 "승자만 선공"이 아니라 **전체 순위가 그대로 다음 판 턴 순서에 반영**되는 방식. 첫 판(이전 순위 기록 없을 때)은 랜덤으로 순서 정함.

- **순위가 동점일 경우**: 동점자들끼리 주사위 하나씩 던져서 **큰 숫자가 나온 사람이 먼저**로 순서 결정

## 랭킹 동점 처리 (확정)

랭킹 화면에서 포인트가 같으면 **승률이 높은 쪽이 위 순위**로 표시됨.

## UI 디자인 (진행 중)

- **비주얼 톤 확정**: "네온 아케이드" — 어두운 배경(다크 네이비~블랙 그라데이션), 주사위 4색(빨강 #ff4d6d·노랑 #ffd93d·초록 #39ff9e·파랑 #4da6ff)을 네온 글로우로 강조. 로고는 "NUM"(흰색)+"LOCK"(초록 글로우) 조합
- **① 로그인 화면**: 4색 주사위 아이콘 → NUMLOCK 로고 → 코드 입력창 → 입장 버튼. 로그인 방식 자체는 기존 확정사항(코드만 입력) 그대로
- **② 메인 화면**: 우상단 프로필 칩(아바타+이름+포인트) → 중앙 로고 → 싱글모드 큰 카드(바로 진입, 파랑/보라 글로우) → 멀티모드는 비활성 카드로 자리만 잡음("준비중" 뱃지, 싱글 안정화 후 활성화 예정) → 그 아래 설정/랭킹/도움말 정사각형 3개(네온 글로우 아이콘, 훌라 서브메뉴와 같은 배치)
- **③ 설정 화면**: 프로필(큰 아바타+이름) → 아바타 선택 그리드(6열) → 접속중 표시 토글 → 저장 버튼 → 싱글/멀티 포인트 스탯 카드(훌라의 "충전" 버튼은 제외 — NumLock엔 충전 개념 없음) → 관리자 모드. 훌라 설정화면 기준으로 만들되 **"승리 문구"는 사용자 요청으로 제외**, **"알림" 토글은 근거 없이 넣었던 항목이라 제외함**(훌라에도 없고 정해진 알림 기능이 없음)
- **④ 도움말 화면**: RULES.md 원문 순서 그대로 압축 — 목표 → 색상별 진행 방향(빨강·노랑 2→12, 초록·파랑 12→2) → 턴 진행 6단계(주사위굴리기/공용행동/추가행동/체크규칙/벌점/줄잠금) → 종료조건+점수표 안내
- **⑤ 랭킹 화면**: 탭을 **싱글/멀티 2개**로 분리(훌라는 훌라왕 탭이 하나 더 있는데 NumLock엔 해당 개념 없어서 제외). 각 행에 승패 기록+승률+포인트 표시(훌라 랭킹과 동일 패턴), 포인트 동점 시 승률로 순위 결정(위 "랭킹 동점 처리" 참고)
- 화면 방향: 로그인/메인/설정/도움말/랭킹은 세로(portrait) 고정, **게임 화면만 가로(landscape)** — 점수판이 11칸짜리 표라 가로 폭이 필요해서 게임 화면만 예외로 확정 (2026-07-20)
- 시안 artifact: https://claude.ai/code/artifact/e9b829e2-3325-4265-8817-b5584d70256e

## 게임 진행 순서도 (확정)

⑥ 게임 화면(진행+정산+한판더, 훌라처럼 한 화면 안의 오버레이 상태들) UI를 만들기 전에, 턴 단위 로직을 먼저 순서도로 정리함.

**1단계 — 턴 시작**
차례인 플레이어가 주사위 6개(흰 2개 + 빨강/노랑/초록/파랑)를 굴림

**2단계 — 공용 행동 (전원)**
흰 주사위 2개의 합 계산 → 모든 플레이어는 원하는 경우 해당 숫자를 한 곳(빨강/노랑/초록/파랑)만 체크 가능, 체크하지 않아도 됨

**3단계 — 추가 행동 (차례인 플레이어만)**
흰 주사위 1개 + 색깔 주사위 1개를 조합 → 해당 색 줄에 체크 가능, 체크하지 않아도 됨

**4단계 — 체크 시 분기: 줄 잠김 조건 확인**
방금 체크한 숫자가 그 줄의 마지막 숫자(빨강·노랑=12, 초록·파랑=2)이고 이미 해당 줄에 5칸 이상 체크되어 있었다면 → 해당 줄 즉시 잠김 + 해당 색 주사위 게임에서 제거(이후 그 색 주사위는 더 이상 굴리지 않음)

**5단계 — 벌점 판정 (차례인 플레이어만)**
2단계와 3단계 모두 체크하지 못한 경우 → 벌점 1개(-5점)

**6단계 — 종료 조건 확인**
줄 2개 잠김 또는 벌점 4개 달성 중 하나를 만족하면, 현재 플레이어의 턴 종료 후 게임 종료. 만족하지 않으면 다음 플레이어에게 턴을 넘기고 1단계로 이동

**7단계 — 게임 종료 → 정산**
각 줄 체크 개수를 점수표에 따라 계산 → 벌점(개수×5점) 차감 → 최종 점수 산출 → 순위 결정 → 1등 보너스 지급(= 1등 점수 − 2등 점수, 1등 동점이면 보너스 없음)

**8단계 — 결과 화면**
표시 항목: 최종 순위, 최종 점수, 획득 포인트, 1등 보너스, 벌점, 각 줄 점수

**9단계 — 한 판 더 / 메인 복귀**
한 판 더 또는 메인으로 돌아가기 선택. 다음 판 턴 순서는 이전 판 최종 순위 순(동점이면 해당 플레이어끼리 주사위 굴려 순서 결정, 첫 게임은 랜덤 순서)

## 게임 화면 UI (진행 중)

위 순서도 기준으로 세부 동작 먼저 확정 후 시안 제작:
- **다른 사람 턴**: 그 사람이 주사위 던지는 순간만 전체화면 포커스(흔들리는 애니메이션)로 전환, 끝나면 기본 상태로 복귀
- **공용 행동 제한시간**: 10초
- **화면 기본 레이아웃**: 평소엔 4명 점수판이 항상 다 보임(남이 뭘 체크했는지 실시간으로 보임) — **4명 전부 실제 숫자로** 2×2 그리드 배치(압축 막대 아님, 다들 어디까지 체크했는지 정확히 보여야 해서). 상단은 얇게 턴 정보+타이머+주사위만. 내 칸/현재 턴인 사람 칸만 테두리 색으로 구분
- **추가 행동 조합**: 흰 주사위 + 색깔 주사위를 직접 탭해서 선택. **한 번 고른 주사위는 확정 — 다시 눌러도 안 바뀜**(같은 종류 주사위를 재선택 불가). 확정한 조합이 이미 체크된 칸이라 쓸 수 없으면, 되돌리지 않고 그대로 이번 추가 행동을 패스 처리
- **추가 행동 제한시간**: 공용 행동과 동일하게 **10초** (시간 내에 주사위를 다 못 고르면 패스)
- **화면 방향**: 점수판이 11칸 표라 가로가 필요해서 게임 화면만 가로로 확정
- ⑥-A(기본 상태), ⑥-B(주사위 굴리는 순간), ⑥-C(추가 행동 — 흰+색 주사위 직접 탭 선택, 선택한 조합 숫자가 내 점수판에 주황 하이라이트로 표시), ⑥-D(정산 결과) 시안 완료
- **인터랙티브 데모 추가**: 정지 화면 갤러리 위에 실제 클릭 가능한 프로토타입도 만들어둠 — 주사위 굴리기→공용행동(10초 타이머+칸 클릭)→추가행동(10초 타이머+주사위 선택 즉시확정+조합 계산)→다른사람턴 보기→게임종료→정산→한판더(준비중 전환)까지 전체 사이클을 직접 눌러볼 수 있음. 잠금/벌점 판정 등 실제 규칙 엔진까지는 구현 안 하고 화면 전환 모션 확인용으로 단순화함. Playwright로 전체 사이클(주사위 잠금 동작 포함) 실제 클릭 테스트 통과 확인함
- 시안 artifact: https://claude.ai/code/artifact/e9b829e2-3325-4265-8817-b5584d70256e
- **⑥-D 정산 결과**: 훌라처럼 결과+한판더 버튼을 한 화면에 합침(별도 "한판더 화면" 없음). 순위별 4줄, 각 줄에 색상별 점수 내역+벌점 소내역, 오른쪽에 최종 점수·1등 보너스·획득 포인트. 지금은 싱글모드 기준이라 "한 판 더" = AI 3명과 바로 재시작. 버튼 순서는 왼쪽 "한 판 더!"(primary), 오른쪽 "메인으로"(ghost)

## 구현 전 결정사항 (확정, 2026-07-20)

- **랭킹 승패 기준**: 훌라와 동일하게 **1등만 승, 나머지는 전부 패**로 카운트
- **신규 유저 초기 포인트**: ~~싱글/멀티 둘 다 0P로 시작~~ → **번복됨, "재화 시스템" 섹션의 충전 방식 참고** (싱글은 100점 시작 + 충전 가능으로 변경)
- **싱글모드 AI 3명**: 훌라와 동일한 캐릭터(돼지/호랑이/강아지) **재사용**
- **잠긴 색 주사위 표시**: 줄이 잠기면 해당 색 주사위는 화면에서 **비활성화 표시**(안 보이게 숨기지 않고, 흐리게/못 씀 표시)
- **재접속/새로고침 복구**: 훌라 싱글모드와 **동일하게** — 연결 끊기면 자동 일시정지, 재접속하면 일시정지 화면에서 재개 버튼으로 이어감. 관리자용 "중단" 버튼도 훌라와 동일하게 제공

## AI 턴 진행 속도 (확정)

- **공용 행동(체크 여부 판단)**: AI는 10초 타이머를 다 기다리지 않고, **1.5초/2초/3초 중 랜덤**으로 딜레이를 준 뒤 체크(또는 패스) 결정을 반영. 매번 같은 타이밍이면 기계적으로 보여서 세 값 중 랜덤으로 선택
- **추가 행동(주사위 조합 선택)**: 딜레이 없이 **즉시** 결정 (차례인 AI만 하는 행동이라 굳이 뜸 들일 필요 없음)
- 참고: 주사위 굴림 애니메이션은 이미 1.1초로 고정돼있음(인터랙티브 데모 기준) → 다 합치면 AI 턴 하나가 대략 3~5초 정도 소요될 것으로 예상

## 구현 전 결정사항 2차 (확정, 2026-07-20)

- **체크 유효성 검증 (중요 — 구현 시 반드시 반영)**: 후보 숫자가 "체크 안 된 상태"인지만 보면 안 되고, **해당 줄에서 이미 체크된 것 중 가장 진행된(오른쪽) 숫자보다 뒤에 있는지**까지 검증해야 함. 예: 빨강 줄에 2,3,5 체크된 상태에서 흰 합이 4가 나와도, 이미 5를 지나쳐서 4는 체크 불가. 인터랙티브 데모에는 이 검증이 빠져있어서 실제 구현 시 새로 넣어야 함(공용 행동/추가 행동 후보 판정 둘 다 해당)
- **가로 강제 불가 문제**: 웹에서는 JS로 화면 회전을 강제할 수 없음(iOS Safari 특히). 게임 화면 진입 시 기기가 세로면 **"화면을 가로로 돌려주세요" 안내 화면**을 별도로 띄우기로 함 (기존 시안엔 없던 화면, 구현 시 추가 필요)
- **공동 1등 승패 카운트**: 훌라와 동일하게 **공동 1등이면 전원 승리로 카운트** (훌라 코드 확인함: `rank === 1`인 사람 전부에게 승 부여하는 방식이라 공동 1등도 둘 다 승리 처리됨)
- **마이너스 최종 점수 처리**: 벌점만 쌓이고 체크를 거의 못 하면 최종 점수가 음수가 될 수 있음(예: 4줄 다 0점 + 벌점4개(-20) = -20점). 훌라처럼 **포인트 지급 시 0에서 바닥을 막음**(`MAX(0, ...)` 방식) — 이래야 "0점일 때만 충전 가능" 조건도 깔끔하게 맞아떨어짐
- **관리자/유저 등록 방식**: 훌라와 **동일** — 코드만으로 로그인하는 구조라 그 코드를 누군가 미리 등록해줘야 함. 설정 화면의 "관리자 모드" 버튼도 훌라처럼 비밀번호로 진입해서 유저 추가/삭제 등 관리 기능 제공
